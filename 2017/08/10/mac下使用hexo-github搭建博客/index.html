mac下使用hexo+github搭建博客

引言

目前公司项目内使用tower管理项目，由于经常会在tower上补充一些项目文档，同时也有一些可以作为个人的技术积累。之前零散的在一些blog网站上写过一些，目前有了github和个人域名之后，想在浩瀚的互联网上有一个自己的小空间。于是开始搭建自己的小窝了。

因为github page可以提供静态页面展示，而blog又不需要太复杂的架构，所以使用github page + markdown 来做blog是个性价比很高的方案。而hexo可以快速将markdown转化成html，并且集成了向github推送的能力，使其成为不错的选择。

参与文章：

  • http://baixin.io/2015/08/HEXO搭建个人博客/
  • http://www.jianshu.com/p/465830080ea9

一、配置需求

Nodejs

Node.js官网下载最新版

Git

Mac下载Xcode自带

github账号

Github官网注册

二、安装hexo

sudo npm install -g hexo

三、hexo初始化

  • 创建一个用作blog的目录

    mkdir ~/Documents/blog

  • 进入到目录中初始化hexo

    hexo init

  • 初始化后会生成如下目录及文件：

    • _config.yml: 配置文件
    • db.json：数据文件
    • node_modules
    • package.json
    • public
    • scaffolds
    • source：md源文件
    • themes：主题文件

四、hexo使用

  • 创建新文章

    hexo n

  • 生成html

    hexo g

  • 本地调试

    hexo s

  • 发布到github

    hexo s

五、hexo的一些配置

配置文件：_config.yml

  • 基本配置
    • title: 网站的title
    • url: 网站的根域名
    • root: 网站的根路径
      • If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/')
  • github关联
    • type: git
    • repo: github上的仓库地址
    • branch：对应的分支名
    • 设置完后，要保存设置
      • npm install hexo-deployer-git --save

六、一些注意事项

  • _config.yml配置中，冒号后面要有一个空格
  • github配置完后，要保存配置 npm install hexo-deployer-git --save
  • 手动编码markdown不太方便，推荐使用markdown编辑器。我选择的是开源的MacDown.关于两大编辑器的链接的文章
